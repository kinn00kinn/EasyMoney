# PayPay銀行 CSV 取込要件

`sample/sample-paypay-bank.csv` をベースに、PayPay 銀行の入出金明細 CSV を取り込む際の仕様を整理する。

## 1. 入力ファイル仕様

- **文字コード**: Shift_JIS（CP932）。UTF-8 の場合でも受け付ける。
- **区切り**: `,`、ダブルクオートで囲まれた RFC4180 互換。
- **想定列**:
  - `操作日(年)` / `操作日(月)` / `操作日(日)`
  - `操作時刻(時)` / `操作時刻(分)` / `操作時刻(秒)`（任意）
  - `取引順番号`
  - `摘要`
  - `お支払金額`（支出）/ `お預り金額`（入金）
  - `残高`
  - `メモ`
- 旧フォーマットの `入出金日`, `出金額`, `入金額`, `内容`, `備考` も引き続きサポートする。

## 2. 正規化ルール

| 項目 | 仕様 |
| --- | --- |
| 日付 | `入出金日` → `取引日` → `日付` の順に使用。存在しない場合は `操作日(年/月/日)` を組み合わせて `YYYY-MM-DD` に整形。時刻があれば 09:00 JST とみなし ISO に変換後、日付部のみ保存。 |
| 摘要 | `摘要` → `内容` → `備考` → `メモ` の順。空の場合は `'PayPay取引'`。 |
| メモ | `メモ` をそのまま格納。 |
| 金額 | 出金系: `出金額` / `出金` / `出金金額` / `お支払金額`。入金系: `入金額` / `入金` / `入金金額` / `お預り金額`。通貨記号やカンマは除去し、円 x100 を整数 (cents) で保持。 |
| 取引方向 | 入金金額が 0 より大きければ `inflow`、それ以外で出金金額があれば `outflow`。両方 0 の場合はスキップ。 |

## 3. 取り込みフロー

1. `POST /api/imports/paypay`
   - ファイルは `multipart/form-data` で送信。
   - Worker 側で Shift_JIS/UTF-8 を判別してテキスト化し、PapaParse で解析。
   - 有効行ごとに `imports`/`import_rows` を作成。
2. UI が解析結果を表示し、カテゴリを選択。
3. `POST /api/imports/:id/confirm`
   - 選択済み行ごとに `transactions` と `entries` を生成。
   - `import_rows` の `status`/`category_id`/`transaction_id` を更新。

これらの要件を満たすことで、サンプル CSV を含む PayPay 銀行の最新レイアウトを確実に取り込める。
